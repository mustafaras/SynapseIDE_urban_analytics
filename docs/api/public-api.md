# Public API Surface

Date: April 23, 2026
Status: Release-candidate reference

This repository is primarily a browser application, so its "API" is the set of stable module entry points and operator-facing integration contracts used across the workbench. The goal of this document is to make those surfaces explicit for maintenance, review, and downstream integration.

## UI Entry Surfaces

| Surface | Entry point | Notes |
|---|---|---|
| Center-panel shell | `src/centerpanel/CenterPanelShell.tsx` | Top-level host for Projects, New Project, Methods, Education, Report, Workflows, Dashboard, and Toolbox. |
| Urban Analytics workspace | `src/features/urbanAnalytics/UrbanAnalyticsModal.tsx` | Modal workspace opened from the IDE shell. |
| Map Explorer | `src/centerpanel/components/MapExplorerModal.tsx` | Primary geospatial rendering and IO surface. |
| Toolbox feature index | `src/centerpanel/Tools/components/CapabilitiesOverviewPanel.tsx` | Release-candidate index for tabs, flows, and runtime labs. |

## Engine Entry Points

| Area | Module entry | Stable exports / contracts |
|---|---|---|
| GeoAI | `src/engine/geoai/index.ts` | Runtime, CV, NLP, model catalog, and hooks. Release-candidate hooks include `useGeoAIStatus`, `useGeoAIModelProfiles`, `useLandCoverClassification`, and `useQueryToSQLRunner`. |
| Streaming | `src/engine/streaming/index.ts` | `createStreamingConnector`, connector types, and `useStreamingConnection`. |
| WASM | `src/engine/wasm/index.ts` | `SpatialIndexWASM`, worker bridge exports, and capability probing through `supportsSpatialIndexWorker`. |
| Spatial DB | `src/engine/spatial-db/index.ts` | DuckDB-WASM-backed spatial querying surfaces. |
| Simulation | `src/engine/simulation/index.ts` | Deterministic analytical engines used by guided flows and dashboards. |
| Spatial statistics | `src/engine/spatial-stats/index.ts` | Regression, autocorrelation, multivariate, interpolation, and spatiotemporal analytical modules. |

## Service Contracts

| Area | Module | Primary contract |
|---|---|---|
| Retrieval-augmented references | `src/services/rag/corpus.ts` | Built-in academic corpus via `BUILT_IN_CORPUS_DOCUMENTS`, `listBuiltInCorpusDocuments`, and `searchBuiltInCorpus`. |
| Retrieval orchestration | `src/services/rag/retriever.ts` | Merges workspace retrieval with the built-in academic corpus. |
| Agent orchestration | `src/services/agents/orchestrator.ts` | `runOrchestrator()` returns planner output plus an explicit `dispatchPlan` generated by the guarded domain router. |
| Agent routing | `src/services/agents/domainRouter.ts` | `buildAutonomyDispatchPlan()` and `summarizeAutonomyDispatchPlan()` provide controlled autonomy hints. |
| Reporting | `src/services/reporting/index.ts` | Templates, citation services, auto-narrative, and report builder UI support. |
| Persistence | `src/centerpanel/lib/persist.ts` | `loadPersistWithReport()`, `migratePersistBlob()`, and `createPersistBlob()` expose versioned persistence with migration notes and deprecation warnings. |
| Browser API wrapper | `src/services/api.ts` | `ApiService`, `ProjectApiService`, and shared response types for optional backend integrations. |

## Status and Runtime Stores

| Store | Module | Purpose |
|---|---|---|
| GeoAI runtime status | `src/stores/useGeoAIStatusStore.ts` | Mirrors model loading, inference, memory, backend, and error state into the status bar and GeoAI surfaces. |
| Streaming runtime status | `src/stores/useStreamingRuntimeStore.ts` | Tracks connector state, received-message counts, throughput, topic metadata, and errors. |
| Spatial index runtime | `src/stores/useSpatialIndexStore.ts` | Tracks spatial index backend and readiness for operator visibility. |
| Map Explorer | `src/stores/useMapExplorerStore.ts` | Controls map visibility, viewport, tools, overlays, and project-state restoration. |

## Event-Level Integration Contracts

| Contract | Shape | Used by |
|---|---|---|
| Cross-panel navigation | `window.dispatchEvent(new CustomEvent("synapse:navigate", { detail }))` | Capabilities overview, right-panel actions, indicator routing, dashboard/report jumps. |
| Status bar bridge | `src/components/StatusBar/statusBridge.ts` | Engine and editor subsystems publish status snapshots into the bottom status bar. |
| Completed runs | `src/stores/useFlowStore.ts` | Guided flows publish reviewable runs that can later be surfaced in Map Explorer and review workflows. |

## Stability Notes

- The browser-side engine modules listed above are the preferred integration points for internal development.
- External provider integrations such as live AI APIs, Google Maps services, live MQTT brokers, and arbitrary WebSocket feeds remain environment-dependent and are documented as operational risks rather than assumed local guarantees.
- Persistence compatibility for legacy challenge fields remains intentionally readable through persist version 5; downstream code should treat those fields as deprecated.
