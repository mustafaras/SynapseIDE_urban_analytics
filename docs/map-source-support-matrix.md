# Map Source Support Matrix

Date: 2026-05-30
Status: Prompt 63 close-out reference

This matrix records shipped Map Explorer source handling, not aspirational GIS parity. A format can be profileable, renderable, queryable, or package-restorable without being safe for every analysis. Missing CRS, license, attribution, vertical datum, noData, source bytes, or provider status remains explicit.

| Format or source | Preflight/profile status | Commit/render status | Query/analysis status | Export/package status | Current limits |
| --- | --- | --- | --- | --- | --- |
| GeoJSON / JSON | Supported | Supported as local vector layer; large files receive render-budget and worker-readiness caveats | Queryable when feature data is present; metric workflows require CRS preflight | GeoJSON and GeoParquet data export supported; small inline sources can be embedded in offline packages | CRS is unknown unless trusted metadata declares it. Large sources may render through sampled/tiled/worker paths. |
| CSV | Supported with coordinate-column preview | Supported after latitude/longitude confirmation | Queryable point layer after commit | Exportable after commit; skipped rows recorded in provenance | Coordinate columns do not prove CRS. Malformed/missing coordinates are counted before commit. |
| Arrow / Feather / IPC | Supported with schema preview, row counts, worker-transfer readiness, and memory estimate | Supported as columnar vector layer | Queryable through worker/SpatialDB-backed paths when published | GeoParquet export supported; package restore preserves lightweight handle metadata | CRS remains unknown unless metadata declares it. Worker transfer is reported separately from map commit. |
| GeoParquet / Parquet | Supported with GeoParquet metadata and schema preview | Supported as columnar vector layer | Queryable through worker/SpatialDB-backed paths when published | GeoParquet export supported; package restore preserves lightweight handle metadata | Declared CRS is copied as source metadata but still requires suitability review before metric analysis. |
| KML / KMZ | Supported | Supported as converted GeoJSON feature layer | Queryable after conversion when geometries are supported | Exportable after commit | CRS is not inferred from coordinates; unsupported KML constructs are not silently fabricated. |
| GPX | Supported | Supported as converted waypoint, route, and track features | Queryable after conversion where geometry and attributes are available | Exportable after commit | Device coordinate semantics, elevation meaning, and CRS remain review caveats. |
| Shapefile `.shp` / `.zip` | Supported | Supported through `shpjs` conversion | Queryable after conversion | Exportable after commit | `.prj` is read when present; missing or unreadable `.prj` leaves CRS missing. Bare `.shp` without sidecars is limited. |
| GeoPackage `.gpkg` | Supported with layer discovery and per-layer CRS extraction when loader metadata exposes EPSG/SRS/WKT fields | Supported; single-layer packages commit directly and multi-layer packages require layer choice | Queryable after selected layer commit | Exportable after commit | CRS is no longer assumed to be EPSG:4326. If browser loader metadata does not expose CRS, the layer stays `missing` and must be declared before metric analysis. |
| FlatGeobuf | Supported as metadata/profile path; URL streaming service exists | Supported for extent-based streaming when provided as an HTTP range-capable URL, not as a generic local full-file commit | Viewport extent queries are supported; total feature count can remain unknown | Source handle can be recoverable by URL; package does not embed remote bytes | Requires HTTP range support. Local `.fgb` upload remains profile-only in the generic import hub. |
| PMTiles | Supported as metadata/profile path and vector-tile source metadata | Supported for PMTiles/vector-tile URL rendering and on-the-fly tiled GeoJSON display | Metric results from tiled/generalized display carry approximation caveats | Source handle can be recoverable by URL; package does not embed remote bytes | Low-zoom geometry is generalized. Precision-sensitive analysis must use full-resolution source geometry. |
| WMS / WMTS / XYZ raster tiles | Supported through external connection metadata | Supported as raster tile overlays when provider URL/capabilities are valid | Not queryable as vector features | Package preserves external-reference metadata, not provider bytes | Environment-dependent: credentials, CORS, rate limits, upstream availability, and attribution remain visible caveats. |
| WFS / OSM / Overpass-style vector services | Supported through external-service path | Supported when the connector returns compatible feature data | Queryable only after features are published into project/map context | Package preserves source reference and unavailable/recoverable state when bytes cannot be embedded | Live success is provider-dependent. Remote catalogs are not silently federated into NL query scope. |
| GeoTIFF | Supported with metadata, CRS, noData, band, histogram, and sampled-pixel preflight | Supported as a sampled image raster layer from the generic import hub; also supported in the raster evidence/inspection panel and terrain DEM source-handle path | Raster QA, sampled histogram, noData, CRS, and terrain sampling are supported; full-resolution raster analytics remain bounded | Package preserves metadata/evidence references; raw large rasters are not silently embedded | Parser samples at bounded resolution and renders a bounded preview, not a full-resolution raster workload. Missing CRS blocks raster analysis readiness. Missing noData is a warning. |
| COG / STAC / Sentinel Hub EO sources | Supported through the Toolbox EO connector stack | Source selection, COG inspection, footprint publication, and downstream GeoAI paths are implemented | Environment-dependent; imported/published results become queryable only through project context | Package/export records provenance and external dependency rather than embedding provider data | Live catalog/process execution requires credentials and upstream availability. |
| CityJSON | Supported as profile/scene source metadata | Supported in the 3D scene path with CityJSON loading and footprint/city-model metadata | 3D scene analysis uses reference metadata and projected/vertical caveats | Package preserves source-handle scene metadata when available | CRS and vertical datum must be declared or caveated. Generic 2D import is not the primary path. |
| 3D Tiles | Supported as metadata/scene source | Supported as 3D Tiles source-handle metadata for scene integration | Scene-level evidence can reference tileset metadata; full tile payload analysis is not claimed | Package preserves external/metadata-only source handle | Vertical datum, tile availability, and external payload access are not inferred. |

## Cross-Cutting Rules

- Planar area, distance, buffer, join-nearest, zoning envelope, and corridor metrics require `CrsPreflight` approval and a projected execution CRS.
- Source handles persist lightweight metadata only. Raw large geometry, DEM cells, 3D tile payloads, external provider bytes, and worker tables are not stored in Zustand.
- Offline packages embed only small inline GeoJSON sources under the package source limit. Large, external, or provider-backed sources restore as precise unavailable/recoverable states.
- Demo, sample, generated, tiled, sampled, user-declared CRS, and metadata-only modes must remain visible in layers, QA, evidence, reports, and exports.
