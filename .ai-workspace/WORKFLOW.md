# AI Development Workflow

## Default Repository
mustafaras/SynapseIDE_urban_analytics

## Standard Workflow
1. Read `.ai-workspace/PROJECT_CONTEXT.md`.
2. Read `.ai-workspace/DECISIONS.md`.
3. Read `.ai-workspace/ARCHITECTURE.md` when technical changes are needed.
4. Create or use a feature branch.
5. Make focused changes.
6. Avoid unrelated edits.
7. Preserve existing functionality.
8. Prepare a pull request.
9. Include risks and test notes.
10. Check GitHub Pages preview when available.

## Branch Naming
Use clear branch names:

- docs/...
- feature/...
- fix/...
- ui/...
- refactor/...
- experiment/...

Examples:

- docs/ai-workspace-context
- ui/refine-main-layout
- fix/file-explorer-actions
- feature/analytics-panel
- experiment/new-dashboard-shell

## Pull Request Summary Format
Every PR should include:

### What changed
Brief explanation of the change.

### Files changed
List of important files modified.

### Risks
Possible side effects or fragile areas.

### Test notes
What was checked or what should be checked locally.

### Follow-up
Recommended next steps.
